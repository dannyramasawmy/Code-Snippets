## Arrays/ Strings / Slicing
```python
# python lists can store any type and act like a stack
arr = []
arr = [0,1,2,3,4,5,6]
arr = ['a','b','c','d','e','f','g']
arr[0]
>>> 'a'
arr[3]
>>> 'd'
arr[0:3]
>>> 'a','b','c'
arr[0::]
>>> arr = ['a','b','c','d','e','f','g']
arr[-3:-1]
>>> 'e','f'
arr[0::2]
>>> ['a', 'c', 'e', 'g']
arr[-1::-2]
>>> ['g', 'e', 'c', 'a']
arr.append('h')
>>> arr = ['a','b','c','d','e','f','g','h']
arr.pop()
>>> arr = ['a','b','c','d','e','f','g']
```

```matlab
% matlab
% numeric array
arr = [0, 1, 2, 3, 4, 5, 6, 7]
% char array
charr = 'abcdefg';
charr(1)
>>> 'a'
charr(4)
>>> 'd'
charr(1:3)
>>> 'abc'
charr(1:end)
>>> arr = 'abcdefg'
charr(end-1:-1:end-3)
>>> 'fed'
% matlab strings are like python strings
% string array 
charr = ["a", "b", "c", "d", "e", "f", "g"]; % strings are objects
charr.append('h')
>>> arr = ["a", "b", "c", "d", "e", "f", "g", "h"]
arr.pop()
>>> arr = ["a", "b", "c", "d", "e", "f", "g"]
```
